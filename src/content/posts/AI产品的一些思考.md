最近一直困在技术、产品、战略的职业选择中，所以索性随心所欲地把我的一些逻辑和想法写在这里。

从长远来看，AI的一个可以预期的产品是将医疗、金融、法律等高昂的专业服务带给普通人。而从近期的话题讨论度来说，VLM也开始逐渐火热起来。但仅从这两个信号我们很难具体得到什么可以执行的目标，前者太漫长，后者刚刚兴起，成熟度与市场接受度都还是未知。

那么又回到我喜欢用的从历史中找规律的方法（或许这是因为一些伟大的事情背后是有规律、有结构的？），从互联网和计算机的角度分别来看待AI接下来的产品发展。

**结论1：满足少部分工作需求，然后才是娱乐/消费市场；这与当下2B火热，2C难以维系一致**

从互联网的角度来看：互联网一开始解决查阅文档的需求，接着把写信变成了更高效的方式邮件。往后AOL 免费光盘轰炸，让更多人接触上网服务（邮件/BBS）。再然后才是电商/门户/UGC等等辉煌时刻。

从这个角度出发理解AI，AI一开始解决获取知识/书写/代码的需求，那么现在的问题是AI时代的杀手应用（邮件）是什么？**这里我想提两个产品：vibe coding和deep research**

另一方面，从工业革命而不只是某一个重要的新技术来看，我们可以参考计算机的发展历史：电子管/军用-晶体管/军用-微处理器/消费主机（1970-1980）。**我认为这是更贴近AI的发展的参考路线**

（1970-1980）普通人（教师、学生、小企业主、极客）主要用它干这几件事：打游戏/学编程/处理文字/记账；【我认为AI正处于这个阶段，包括企业采购CC/codex/cursor等】

（1984-1995）少部分精英开始使用GUI来进行文字排版（取代了打字机）；后随着PC价格的降低、办公套装的成熟，普通人开始使用电脑（取代了计算器和账本）

1995年之后的事情我们都知道，迎来了互联网

**结论2：从两个角度来看，无论AI作为计算机新技术，还是作为一场新的革命，都已经走到甚至走完了第一层，它已经解决了一部分人的工作需求**（写代码/文字工作/学习知识/画图/情感陪伴）。**接下来它要走到它真正证明自己的第二层：去取代某种方式，去取代某种机器，甚至可能是取代某些人**（获取知识和降低执行成本）

AI从一开始的CV/NLP作为一种高效的机器学习方法，到GPT的出现开始泛化或者逐渐应用到更多的场景（写代码/文字工作/学习知识/画图/情感陪伴），总的来说让知识的获取（LLM）以及执行（Agent）变得更加容易了。如果要说取代了什么，那就是相当一部分执行层（白领、初级的老师、像我这样的知识传播者等等），即不涉及到判断和目标制定，只是按照SOP执行的人。

进一步的抽象需求可以说AI让探索与验证变得更加容易，但我说不出这个具体的更通俗的操作是什么。我模糊地感觉到AI成为了一个良师益友，回答了对世界的困惑、对自身想法的不确定、跨界时的困难。我曾想把它总结为**AI改变的基本需求不在于（获取知识和降低执行成本），而在于了解认识自己，认识世界，获取外部反馈**，就像Rosebud、PillowTalk、Clio、Sort AI等产品。但我认为这个概括太虚，可能是某个阶段的概括，而不是某个杀手应用的描述。

回到杀手应用，按照我之前的预测，2026年应该是ChatGPT出现后市场蓬勃的一年，在这一年出现了OpenClaw，所以**我个人坚定地相信Agent就是“邮件”。但具体说来两个产品**（deep research、vibe coding/auto research）我都觉得有些问题，或者市场没有完全认可他们。这背后可能有两个因素：**替代成本和自发动机**

* 写信除了在用于商业交流外，还可以以更加私人、更加自发的形式存在。而deep research和auto research两者需要的自驱力门槛都更高，一个具体的表现就是这两者往往出于工作需要，很少出于生活需要或者好奇心。

* 随着而来的是成本的可感知，对于写信而言，一封信从写完到寄出再到收到回信，这个过程是漫长的（一到两周），而email的出现让这个过程变得一天内可以实现。这个成本优化是完全可感的。而deep research和auto research虽然也能缩短对应的时间，但这个过程有时就是人们的日常工作，把无穷长的链条缩短成二分之一无穷长；加之互联网本来将搜索成本降的很低，麻烦的地方在于整理知识，所以成本优化是更隐蔽的。

写到这里，我在想两件事：1、我写的太虚，有没有可能大家并不是需要解放一个新的需求；**只是需要把deep research、vibe coding这种重的产品变成简单的workflow agent来完成工作生活中的琐事**；2、如果有新的需求，这个需求是什么？**是认识自己，认识世界，获取外部反馈？还是某种我们平时在做、有自驱的、可以高频进行的动作？**

**结论3：可以明确的是未来发展趋势有：把AI变得更便宜，skill更成熟（私教/个人助理），把获取知识和执行成本进一步降低，让普通人感受到**

不用问AI的好处就是把知识成本和执行成本变薄，这对专家来说是好事，但普通人能做什么？事实上只要AI足够火，那么FOMO心理就会让大家购买AI产品，而不是真的为了什么具体而实际的目标，这和当年PC的市场是一个道理。

**结论4：技术层面可以认为：便宜/私教/助理，也就是推理/路由/长期记忆/垂直场景/延迟**

* 便宜
  
  * 推理成本：模型蒸馏、量化稀疏推理、KV缓存、长对话优化、投机解码
  
  * 大模型与超长上下文：记忆外置，尽量用小模型做 routing，只在必要时调强模型

* 私教/助理
  
  * 持久化个人记忆：记忆抽取、记忆更新和遗忘
  
  * 长期对齐：防止漂移、用户级微调、基于 prompt + memory 的动态 system prompt 
  
  * 任务拆解：具体的skill；多步推理稳定性、工具schema对齐、错误回滚重试
  
  * 主动推送
  
  * 纠正反馈
  
  * 隐私

**总结：提出了三个问题，解决了一个半：**

* 产品要求（便宜/私教/助理；推理/路由/长期记忆/垂直场景）

* AI之于互联网，到底把什么现在的需求变得简单（回答了一半）
  
  * 我认为人不是无时无刻在工作，所以现在的产品不是答案，是过程；但一个可能的方向是能力化与轻量化
  
  * 我心目中的答案有两个可能
    
    * 一是现在需求的高效实现、高频替代
    
    * 一个是隐秘需求的被正视

* VLM到底有什么用（游戏；导购；多模态=多媒体=<u>先让现在的agent产品变好</u>，未来才是更多的形态）

**不足：必须承认以上是2B，关于2C我会探索但我没有想法，我认为是技术基础还不够成熟，而不是产品形态/商业模式不够好**
